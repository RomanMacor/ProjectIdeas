Requirements Specification

1      Introduction

1.1  Aim

To develop a web application using Java EE technologies that would allow companies and individuals to insert project ideas into the system. Students could then choose one of these ideas as their final year project.

1.2  Key personnel

Responsible owner

Jim Briggs

Design authority

Roman Macor

1.3  Rationale for development

One implementation of the system is running on http://www.sums.port.ac.uk/pdb/, but does not seem to be actively used. It is not implemented in MVC style which makes it difficult to maintain and adjust to possible changes in the future.

Functionality could be significantly improved. After a project idea is submitted it can not be edited or deleted and every time the same person is creating new idea they also have to fill in all the details about the company. This could probably be fixed by registration of the users who wish to submit project ideas.

1.4  Interactions and dependencies

Existing data can be accessed by following instructions from http://briggs.myweb.port.ac.uk/WEBP/cases/pdb/db.htm, however new database needs to be created in order to enable editing and deleting of the project ideas.
